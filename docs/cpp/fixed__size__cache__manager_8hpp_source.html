164 /**
165  * \ingroup fileio
166  *
167  * A global singleton object managing the allocation/deallocation
168  * of cache blocks. The basic mechanism of operation is such:
169  *
170  * - For every new cache block requested:
171  * - If there is FILEIO_MAXIMUM_CACHE_CAPACITY_PER_FILE free bytes,
172  * a new cache block of FILEIO_INITIAL_CAPACITY_PER_FILE is allocated, where
173  * the new cache block is permitted to grow up to
174  * FILEIO_MAXIMUM_CACHE_CAPACITY_PER_FILE. The capacity is not charged as utilization
175  * until it is actually used. i.e. utilization is only incremented by
176  * FILEIO_INITIAL_CACHE_CAPACITY_PER_FILE. Then as more memory is allocated for the
177  * cache, then utilization is incremented again.
178  * - If there is < FILEIO_MAXIMUM_CACHE_CAPACITY_PER_FILE free bytes available:
179  * The largest cache block is evicted. If there is
180  * FILEIO_MAXIMUM_CACHE_CAPACITY_PER_FILE free bytes, Goto the first case.
181  * Otherwise, create a new cache block with all the remaining free bytes.
182  *
183  * The relevant constants are thus:
184  * FILEIO_MAXIMUM_CACHE_CAPACITY : the maximum total size of all cache blocks
185  * FILEIO_MAXIMUM_CACHE_CAPACITY_PER_FILE : the maximum size of each cache blocks
186  * FILEIO_INITIAL_CAPACITY_PER_FILE : the initial size of each cache blocks
187  *
188  * Overcommit Behavior
189  * -------------------
190  * We try our best to maintain cache utilization below the maximum. However,
191  * it is possible to exceed maximum cache utilization under certain race
192  * conditions since we avoid locking on the cache utilization counter.
193  */
