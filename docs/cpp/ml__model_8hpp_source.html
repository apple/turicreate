21 /**
22  * ml_model model base class.
23  * ---------------------------------------
24  *
25  * Base class for handling machine learning models. This class is meant to
26  * be a guide to aid model writing and not a hard and fast rule of how the
27  * code must be structured.
28  *
29  * Each machine learning C++ toolkit contains the following:
30  *
31  * *) state: This is the key-value map that stores the "model" attributes.
32  * The value is of type "variant_type" which is fully interfaced
33  * with python. You can add basic types, vectors, SFrames etc.
34  *
35  *
36  * *) options: Option manager which keeps track of default options, current
37  * options, option ranges, type etc. This must be initialized only
38  * once in the set_options() function.
39  *
40  *
41  * Functions that should always be implemented. Here are some notes about
42  * each of these functions that may help guide you in writing your model.
43  *
44  * *) name: Get the name of this model. You might thinks that this is silly but
45  * the name holds the key to everything. The unity_server can construct
46  * model_base objects and they can be cast to a model of this type.
47  * The name determine how the casting happens. The init_models()
48  * function in unity_server.cpp will give you an idea of how
49  * this interface happens.
50  *
51  * *) save: Save the model with the turicreate iarc. Turi is a server-client
52  * module. DO NOT SAVE ANYTHING in the client side. Make sure that
53  * everything is in the server side. For example: You might be tempted
54  * do keep options that the user provides into the server side but
55  * DO NOT do that because save and load will break things for you!
56  *
57  * *) load: Load the model with the turicreate oarc.
58  *
59  * *) version: A get version for this model
60  *
61  *
62  */
